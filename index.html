Git Out

Get Better at Using Git

Version control is really useful to keep track of changes to scripts or any kind of text file. Changes are tracked on a line-by-line basis, so you can travel back to snapshots in the life of the text. Two people can edit the same file at the same time, and as long as they're changing different lines of text, version control can merge it all together.

And then you edit the same line of text, or realize you were editing an old version, or a file name changed, or ... a lot of other things. And it's all a mess. Most version control systems have ways to deal with these things, but I always forget how.

This site has notes on how to use the command line version of git to either avoid problems or fix problems that I run into.

What do you need to do?

Select from the following.

Start working on a new thing

Create default .gitignore

mkdir ~/.git_templates

git config --global init.templatedir ~/.git_templates

nano ~/.git_templates/.gitignore

Creates a hidden folder to hold templates, configures git to use that folder, and then adds a .gitignore

mkdir
create a directory
~/.git_templates
the directory that will hold the git templates, can be named and located somewhere else
git config
access the configurable settings of git
--global
settings that affect all repositories on your machine
init.templatedir
the location of the templates used when a git repository is first created
nano
a command line text editor, other editors can be used like vi or subl
~/.git_templates/.gitignore
a file used to define what files git should ignore

Start a new git repository

mkdir repo-name

cd repo-name

git init .

Creates a new directory and then initializes git versional control for the contents of that directory.

mkdir
create a directory
destination
the directory that will hold the git repo
cd
move terminal into the directory
git init
initialize the directory as a git repository by creating the hidden folder that track versioning
